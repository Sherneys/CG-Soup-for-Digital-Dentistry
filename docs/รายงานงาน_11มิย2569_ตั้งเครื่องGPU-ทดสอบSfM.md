## 2. ติดตั้ง COLMAP (CUDA build)

- **COLMAP 4.1.0.dev0** (release tag 4.0.4) ติดตั้งที่ `D:\Tools\colmap` — เพิ่มเข้า user PATH แล้ว
- ทดสอบ GPU จริง: `feature_extractor` ขึ้น `Creating SIFT GPU feature extractor` ✅
- ⚠️ **ข้อควรจำ:** COLMAP 4.x เปลี่ยนชื่อ option — ใช้ `--FeatureExtraction.use_gpu` / `--FeatureMatching.use_gpu` (ชื่อแบบ 3.x เช่น `--SiftExtraction.use_gpu` ใช้ไม่ได้แล้ว)
- โบนัสเวอร์ชันนี้: มี **ALIKED** (learned features) ในตัว — ตัวเลือกสำรองถ้า SIFT จับ feature บนผิวหน้าเรียบได้น้อย

## 4. SfM Pipeline (ไฟล์ใหม่: `src/sfm_pipeline.py`)

สคริปต์รัน COLMAP อัตโนมัติครบทุกขั้น: `feature_extractor → exhaustive_matcher → mapper → image_undistorter → patch_match_stereo (CUDA) → stereo_fusion → poisson_mesher`

ผลกับชุด buddha (67 ภาพ):

| ขั้น | ผล |
|---|---|
| Dense fusion | **1,696,937 points** (`output/buddha_sfm/dense/fused.ply`) |
| Poisson mesh | 4.28M vertices / 8.55M faces (`mesh_poisson.ply`, 208MB) |
| Simplify (COLMAP `mesh_simplifier`, ratio 0.02) | **89,258 vertices / 170,987 faces** (`mesh_coarse.ply`) |
| เวลารวมทั้ง pipeline | ~นาทีระดับเดียวบน 4070 Ti (dense stereo เร็วมาก ~0.6–1.0s/sweep) |

## 5. Curvature/Density Module กับ mesh จาก SfM จริง

- รัน `src/curvature_density.py` กับ `mesh_coarse.ply` → ผ่าน
- **ผลตรงเกณฑ์ S1:** บริเวณรายละเอียดใบหน้า (จมูก ปาก หู ตา) "ร้อน" / ฉากหลัง-ระนาบแบน "เย็น" — ดู `output/mesh_coarse_density.png` + `.ply` แบบมีสี
- ตัวเลข: curvedness median = 11.32, density median = 0.3838

## 6. แก้บั๊ก: degenerate vertex ใน `curvature_density.py`

- **อาการ:** `RuntimeWarning: invalid value encountered in divide` ตอนรันกับ mesh จาก SfM จริง (mesh สแกนมี degenerate vertex ที่ normal เป็นศูนย์/NaN) — เดิมรอดเพราะ NaN ทำให้ `lstsq` พังแล้วถูก `try/except` จับ**โดยบังเอิญ**
- **การแก้:** guard ตรง ๆ ใน `principal_curvatures()` — ข้าม vertex ที่ normal ไม่ finite และข้ามเมื่อ tangent vector norm ≈ 0
- **ยืนยัน:** รันซ้ำด้วย `-W error::RuntimeWarning` → ไม่มี warning และตัวเลขผลลัพธ์เหมือนเดิมทุกค่า

## บทเรียน/ข้อสังเกตสำหรับงานหน้าจริง

1. **Poisson สร้างผิวหลอน**บริเวณฉากหลัง/ขอบ — ตอนทำหน้าจริงควร crop ด้วย bounding box หรือ mask ก่อนเข้าโมดูล curvature
2. Mesh จากสแกนจริง**ไม่สะอาด**เท่า mesh สาธารณะ (degenerate vertices) — โมดูลตอนนี้กันไว้แล้ว
3. ความเร็ว GPU เหลือเฟือ: ภาพชุด A จริง 40–80 ภาพจะใช้เวลาประมาณเดียวกับ buddha (หลักนาที)
